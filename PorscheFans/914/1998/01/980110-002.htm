The "winmail.dat" files are attachments that Microsoft email programs
*can* be switched off at the client level. Nick, it is not an IS Server
problem - your SMTP or POP3 server (even Microsoft Exchange (v1 - 4))
will only send what your client system sends - it won't attach
winmail.dat files...   Without going into too much technical detail -
the attachments originates from the client end rather then the server.
So, try the following:

If you are using Internet Mail with I.E. 2.X
Go to "My Computer" or Settings from the "Start" menu;
Open the control panel;
Open "Mail & Fax" Icon;
Double click on your current profile if appropriate;
Select your "Internet Mail" settings;
The "General" tab will appear - select "Message Format" in the Mailbox
dialog area;
Deselect (or unselect) "Use MIME when sending messages"
Hit the "OK" button, then the "Apply" button.
Save your changes by pressing "OK"

If you are using Internet Mail with I.E. 3.0 - 3.02
Open Internet Mail;
Go to "Mail" on the menu bar;
Select  "Options";
The first tab are the "Send" options;
In the Mail Sending format, select "Plain Text, then hit the "Settings"
button next to Plain Text
message format, select "Plain Text" - encoding should be MIME in text
using "None" format.

If you are using Outlook Express with I.E. 4.0
Open Outlook Express;
Goto "Tools" - "Options" and click on the "Send" tab;
Again, change your send settings to "Plain Text" - encoding should be
Uuencode.

If you have any problems or are using another version, e-mail me with
what you have... Hope this helps you out...
