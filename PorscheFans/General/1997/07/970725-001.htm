>Patrick,

>Actually, an IPX and a 128KB ISDN could easily handle a 2000 user mailing
>list. For God sakes, if all the IPX had to do was handle a mailing list,
>it will be not a very busy host. Yes, sendmail will be very busy but not
>all the time.....



Actually, no. To run a large mailing list efficiently, you need to split
the outgoing queue into many small groups and fork a sendmail process for
each so that they run in parallel. Otherwise, sendmail delivers serially
and it will take forever to deliver messages to a large (~2000) member list.

Given the traffic on the 924-944 list alone, you'd probably have between 50
and 100 jobs in the queue at any given time during peak (8-5) hours. Your
IPX would be OK if you could put enough memory in it, but those old
machines don't hold much RAM -- 64MB max if I remember correctly. Trust me
-- I used to run list of similar size and activity on a SPARC 2 and it was
fairly taxing on the hardware.

The bandwidth wouldn't be a problem except for when the digests are
distributed, at which time you would easily saturate a 128k link.

There are enough sysadmins on this list that we could certainly find the
resources to run Porschefans efficiently, but the real question is whether
we have the right to run it in the first place. What if the current
listmaster -- assuming he can be located, or will answer his mail --
doesn't want to give up the list? Surely we're not talking about a listserv
coup here, are we?
