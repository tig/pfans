Indeed, the "burst" command in the MH mail reader for UNIX will take
an RFC 1153 compliant digest and break it into individual messages in
your "inbox".  I find this very useful if I want to save or reply to
one of the messages inside a digest.

In addition, the mh-e Emacs front end for MH also has a command that
will skip ahead to the next message in a digest without breaking it
apart.  This is useful if you have read the first two sentences of a
message but now find yourself wanting to go to the next message.

When people asked about tools for splitting digests, I didn't respond
because PFiles digest format isn't quite RFC 1153 compliant, which is
what originally  prompted my email to the list master.

I hope you found this useful, but I suspect that there are few enough
UNIX users out there that most of you didn't.

For more information about MH:  http://www.ics.uci.edu/~mh/
For more information about mh-e: http://www.x.org/misc/mh-e
